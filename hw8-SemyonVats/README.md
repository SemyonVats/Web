# Техническое задание

## Подготовка проекта

1. Запустите проект:
   - Поправьте `profile.properties`
   - Используйте базу данных вида `uXX`
2. Запуск создаст таблицу `user`
3. Добавьте к таблице колонку `passwordSha` со значением по умолчанию `NULL`
4. Убедитесь, что всё работает:
   - Регистрация
   - Вход
   - Выход
   - Просмотр пользователей

**Ссылка на базу данных:** http://wp.codeforces.com/phpMyAdmin/

**Сборник документаций:** [docs list](./docs/DOCS.md)

## Задача 1: Страница профиля пользователя

- Создайте страницу профиля пользователя по ссылке `/user/{id}`
- Используйте параметр в методе контроллера с аннотацией `@PathVariable`
- Отобразите данные пользователя (`id`, `login`, `creationTime`) или надпись "No such user" если пользователь не существует
- Исправьте отображение логотипа и иконок языков на странице со списком пользователей

## Задача 2: Сущность Notice

- Добавьте сущность `Notice` с полями:
  - `id`
  - `content` (используйте аннотацию `@Lob`)
  - `creationTime`
- Заметка: `Notice` не имеет связи с `User` (пока без связей)
- Создайте:
  - `NoticeRepository`
  - `NoticeService`
- В классе `Page` добавьте метод `getNotices` (аналогично `getUser`) с аннотацией `@ModelAttribute`
- Все потомки `Page` должны добавлять в модель переменную `notices`
- Измените `commons.ftlh`:
  - Замените блок "Pay Attention" на серию блоков, отображающих все notices
- Добавьте несколько notices через базу данных
- Убедитесь, что на всех страницах отображаются notices в сайдбаре

## Задача 3: Форма добавления Notice

- Добавьте форму для создания `Notice`
- Форма должна содержать:
  - Одну `<textarea>` для содержимого
  - Кнопку "Add"
- Реализуйте валидацию вводимых данных

## Задача 4: Блокировка пользователей

- Добавьте пользователям колонку/поле `disabled` (по умолчанию `false`)
- На странице `/users` добавьте колонку с действиями
- В каждой строке таблицы разместите одну кнопку (доступ только для админов):
  - `Disable` для активных пользователей
  - `Enable` для заблокированных пользователей
- Реализуйте функциональность:
  - При нажатии кнопки отправляется форма, устанавливающая новый статус пользователя
  - Семантика обработки: "установить заданный статус", а не "сменить статус на противоположный"
  - После обработки формы страница перегружается с обновленными статусами
- Добавьте проверку на странице входа: при попытке входа заблокированного пользователя сообщение "User is disabled"

## Бонус 1

В задании 4 вместо использования в каждой строке таблицы отдельной формы используйте 1 форму и ее заполнение при помощи `JavaScript`.

## Бонус 2

Добавить базовое тестирование (механизм тестирования использовать [именно фреймворка](https://docs.spring.io/spring-framework/reference/testing/unit.html)).
Покрыты тестами должны быть все классы, содержащие логику.
Классы стоит сделать более тестируемыми (использовать [dependency injection](https://habr.com/ru/articles/350068/)).
