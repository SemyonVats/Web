# HW11

## Подготовка проекта

### Загрузка и настройка
1. Скачайте проект из приложенного сообщения
2. Перейдите в директорию `frontend` и выполните:
   ```bash
   npm install
   ```
3. Запустите frontend-приложение (аналогично предыдущему ДЗ):
   ```bash
   npm run start
   ```

### Настройка базы данных

**Ссылка на phpMyAdmin:** http://wp.codeforces.com/phpMyAdmin/

1. В директории `backend` настройте базу данных в файле `profile.properties`
2. Добавьте пользователей и посты в БД
3. Установите пароли пользователям с помощью SQL-команды:
   ```sql
   UPDATE user 
   SET passwordSha = SHA1(CONCAT('1be3db47a7684152', login, password)) 
   WHERE id = ?id
   ```
   Где:
   - `login` - логин пользователя
   - `password` - пароль пользователя
   - `id` - идентификатор пользователя

## Задача 1: Страница со списком пользователей
- Реализуйте endpoint в бэкенде: `GET /api/users`
- Во frontend используйте код из предыдущего ДЗ для отображения списка пользователей

## Задача 2: Защита роутов и обработка 404
- Запретите неавторизованным пользователям просматривать страницу со списком пользователей
- Реализуйте 404 страницу для обработки неизвестных URL
- Используйте возможности `react-router-dom` для редиректа на 404 страницу

## Задача 3: Главная страница с постами
- На главной странице (Index) отобразите все посты
- Переиспользуйте код из предыдущего ДЗ для отображения постов
- Используйте разметку article из предыдущих заданий
- Отсортируйте посты в хронологическом порядке (от новых к старым)

## Задача 4: Регистрация через REST
- Перенесите форму регистрации из предыдущего задания во frontend
- Реализуйте endpoint в бэкенде: `POST /api/users`
- После успешной регистрации:
  - Автоматически аутентифицируйте пользователя
  - Выполните редирект на главную страницу (Index)
- Добавьте валидацию данных на стороне клиента и сервера

## Задача 5: Создание постов через REST
- Реализуйте endpoint в бэкенде: `POST /api/posts`
- После успешного создания поста:
  - Инициируйте перезагрузку данных постов в компоненте App
  - Обновите сайдбар и главную страницу без перезагрузки
- **Бонус:** Реализуйте "live" режим отображения постов:
  - Новые посты должны появляться без перезагрузки страницы
  - Используйте WebSockets или периодический опрос сервера

## Задача 6: Просмотр отдельного поста
- Реализуйте страницу просмотра отдельного поста по URL: `/posts/:id`
- Добавьте переход на эту страницу по клику на "View All"
- Отобразите:
  - Полный текст поста
  - Информацию об авторе
  - Дату создания
- Используйте параметр маршрута `id` для идентификации поста
