## Бонус 1
В реальных проектах никто не станет использовать булевый флажок для определения роли пользователя.
Для определения ролей пользователя могут использоваться различные механизмы.
Один из гибких способов настроить роли пользователей - `RBAC` (Role-based access control).

В качестве бонусного задания предлагается реализовать его простую версию:
- Создайте иерархическую структуру из сущностей:
  - `Role` - пользовательская роль. Содержит некоторое количество `Privilige`. Связана `Many-toMany` с `User` 
  - `Privilige` - "право доступа". Регулирует один конкретный доступ (например, доступ к редактированию постов). Связана `Many-to-Many` с `Role`.
- Продумайте структуру базы и создайте необходимые таблицы
- Создайте набор `Role` и `Privilige`, присвойте их пользователю вместо булевого флага
- Продемонстрируйте корректную работу

Простой пример использования:
```java
final Privilige canCreatePost = new Privilige("post editor");
final Privilige canDeleteUsers = new Privilige("users deleter");
final Role adminRole = new Role("admin", List.of(
        canCreatePost,
        canDeleteUsers
));
final Role writer = new Role("writer", List.of(
        canCreatePost
));

final User writerUser = new User("username", writer);
final User adminUser = new User("admin_username", admin);
```
