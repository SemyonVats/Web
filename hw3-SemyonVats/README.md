## Задача 1

***Задание:***

В настоящий момент, если делать правки в статике прям в IDEA, то по F5 ресурсы не обновляются. Модифицируйте `StaticServlet` таким образом, чтобы он сначала смотрел в нужную подпапку `src` вашего проекта, и, если файл найден, то возвращал его.

***Указания:***
- `getServletContext().getRealPath("/static" + uri)` должен вызываться во 2-ую очередь, после попытки найти файл в соответствующей подпапке в `src`.
- Убедитесь, что невозможно получить из `StaticServlet` файлы, помимо этих 2-х директорий (например, `uri = './../../../<file path from outer scope>'`)
- Убедитесь, что проходят все три теста из `TestsA`.

## Задача 3

***Задание:***

Ваша задача добиться работоспособности страницы http://localhost:8080/messages.html
Для этой страницы написан фронтенд, всё что осталось реализовать — один сервлет для обработки запросов.
Должны корректно обрабатываться следующие запросы (все запросы передаются методом `POST`, возвращают результат в `JSON`):
1. `/message/auth`: ожидается опциональный параметр `user`, содержащий имя текущего пользователя. Если он задан, необходимо сохранить его в сессию. Вернуть необходимо авторизованного пользователя или пустую строку (если никто не авторизован).
2. `/message/findAll`: возвращает все сообщения в виде массива объектов с ключами `user` и `text`. Сообщения необходимо возвращать в порядке создания (подумайте, какое дополнительное поле стоит создать для такой сортировки). Пример валидного ответа: `[{"user":"mike","text":"test"},{"user":"lena","text":"hi"}]`.
3. `/message/add`: ожидается параметр `text`. Запрос добавляет сообщение от имени текущего авторизованного пользователя с заданным текстом.

***Указания:***

Не забывайте о том что работа вашего сайта (ввод через формы, вывод сообщений) должна быть безопасной.

Для вывода результата всюду используйте "`Content-Type: application/json`".

Для сериализации объекта в JSON используйте библиотеку `Gson`.
Для этого добавьте зависимость в `pom.xml` от артефакта `groupId=com.google.code.gson, artifactId=gson, version=2.9.1`.

Пример сериализации:
```
String json = new Gson().toJson(objectToConvert);
response.getWriter().print(json);
response.getWriter().flush();
```


## Задача 4

***Задание:***

Написать фильтр `CaptchaFilter`, который будет перехватывать все `GET`-запросы, если в текущей сессии не отмечено, что пройдена каптча.

Если капча не пройдена, фильтр должен загадать случайное число от 100 до 999, записать это в сессию (ожидаемый ответ).
На фронтенде должна быть отрисована картинка с ожидаемым кодом и форма для его ввода.
Картинку по тексту можно сгенерировать с помощью `ImageUtils.java`.

Пользователь отправляет форму, фильтр ловит ответ и, если ответ совпал с ожидаемым, в сессии отмечается, что капча пройдена и больше она не отправляется.
Иначе фильтр перезагадывает число и опять показывает форму.

Фильтр `CaptchaFilter` нужно добавить в `web.xml` для перехвата всех запросов.
