## Задача 1: Сущность Comment и функциональность комментариев

##### Добавление сущности Comment
- Создайте сущность `Comment` с полями:
  - `id`
  - `text` (используйте аннотацию `@Lob`)
  - `user`
  - `post`
  - `creation_time`

##### Настройка связей
- Добавьте `List<Comment> comments` в сущность `Post`
- Настройте двунаправленный мэппинг:
  - `Many-to-one` и `one-to-many` для отношения комментарий-пост
- Настройте однонаправленный мэппинг:
  - `Many-to-one` для отношения комментарий-пользователь
  - Комментарий знает своего пользователя, но у пользователя не хранятся комментарии

##### Функциональность
- Создайте страницу для отображения поста и комментариев по URL `/post/{id}`
- Отображайте комментарии в хронологическом порядке
- Для авторизованных пользователей добавьте форму добавления комментария (`textarea` для текста и кнопка отправки)
- Измените главную страницу: заголовки постов по клику должны вести на страницу индивидуального поста

## Задача 2: Сущность Tag и отношение многие-ко-многим

##### Добавление сущности Tag
- Создайте сущность `Tag` с полями:
  - `id`
  - `name`

##### Настройка связи с Post
- Реализуйте отношение многие-ко-многим между `Tag` и `Post`
- Сделайте однонаправленный мэппинг:
  - Добавьте поле `Post#tags`
  - Реализуйте связи по образцу связей пользователей и ролей

##### Инициализация данных
- Добавьте несколько тегов вручную (записи в таблицу `tag`)
- Добавьте связи между постами и тегами (записи в таблицу `post_tag`)
- Отображайте список тегов при показе поста

## Задача 3: Добавление тегов в форму создания поста

##### Расширение формы
- Добавьте поле `Tags` в форму создания поста
- Реализуйте ввод тегов через пробельные символы (whitespaces)

##### Валидация
- Выполняйте проверку тегов после разбиения на слова:
  - Теги должны состоять только из латинских букв
  - Выбрасывайте ошибку валидации для некорректных тегов

##### Сохранение
- При сохранении нового поста:
  - Сохраняйте пост вместе с тегами
  - Добавляйте новые теги в таблицу `tag` при необходимости
  - Используйте существующие теги, где возможно (тэги должны быть уникальны)
